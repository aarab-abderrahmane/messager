Features — MSN Chat
Everything You Loved, And More

All the nostalgic WLM features, rebuilt with modern full-stack technology.

Real-time Messaging
💬
Real-time Messaging
Instant message delivery via native WebSocket. Typing indicators, message history on connect, and live user presence tracking.
GIF & Sticker Picker
🎭
GIFs & Stickers
Giphy API integration with a searchable GIF drawer, plus a curated local sticker collection — just like Winks & Emoticons.
Voice Messages
🎙️
Voice Messages
Record and send voice clips via the Web Audio API. Server-validated with 1MB limit and strict audio/ header checks.
PDF Sharing
📄
PDF Sharing
Send up to 3 PDFs at once with inline preview thumbnails. Server validates file headers, size (2MB limit), and sanitizes filenames.
Photo Sharing
📸
Photo Sharing
Share images with fullscreen preview dialogs, zoom controls, and pending upload confirmation. 2MB size limit enforced server-side.
Reactions & Replies
😍
Reactions & Replies
React with emoji (one per user, toggle off/on) or thread a reply. Reactions broadcast in real-time to all connected clients.
Themes
🎨
Themes
Switch between multiple WLM-inspired colour themes with font, color, and font-size customization applied globally.
Online Contacts Panel
👥
Online Contacts
Sidebar showing who's online/offline with live status updates — powered by server-side USER_STATUS_UPDATE broadcasts.
User Profiles
👤
User Profiles
Editable profile with 30+ classic MSN avatars. View other users' profiles with creation date, IP info, and status details.
Gifts
🎁
Gifts
Send animated gift messages to friends — a modern take on the classic MSN Wink feature with visual flair.
Text Formatting
✏️
Text Formatting
Rich text input with formatting toolbar — bold, italic, and more styling options before sending your messages.
Breaking News Panel
📰
Breaking News
Live news panel in the sidebar with expandable article cards, cover images, attachments, and custom scrollbar styling.

🛡️ Server Middleware Pipeline

🔄 Connection Lifecycle

WebSocket Flow 
Client connects → WS Server assigns rate-limit tokens
Client sends AUTH {token} → Server validates → Sends HISTORY + AUTH_SUCCESS
Server broadcasts USER_STATUS_UPDATE to all clients
Client sends messages → Validated → Stored → Broadcast to all
Client disconnects → Removed from clients → Status re-broadcast
Getting Started — Developer Setup
Developer Setup

Get MSN Chat running locally — both backend and frontend.

1. Clone the repository

bash 
git clone https://github.com/aarab-abderrahmane/messager.git
cd messager

2. Backend setup

bash 
cd server
npm install

# Create .env from example
cp .example.env .env
server/.env 
PORT=5000               # Express + WS server port
MAX_TOKENS=10           # Rate limiter token bucket size
JWT_SECRET=mysecretkey  # Auth secret

3. Frontend setup

bash 
cd ../frontEnd
npm install

4. Run both servers

bash 
# Terminal 1 — Backend
cd server && npm run dev   # starts on http://localhost:5000

# Terminal 2 — Frontend
cd frontEnd && npm run dev # starts on http://localhost:3000
